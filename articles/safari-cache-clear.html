Safariでキャッシュを削除する方法（iPhone/iPad）

ページが真っ白、表示が崩れる、ログインできない、動作が重い…
そんな時は「キャッシュ（Webサイトデータ）」を整理すると直ることがあります。

まずは最短で直す（おすすめ手順）

迷ったらこの順でOKです。

最短 これで直ることが多い

① Safariを終了して開き直す
アプリ切り替え画面でSafariを上にスワイプして終了 → 再起動。

② 直らない時だけ“データ削除”
影響大なら「方法2（サイトだけ削除）」が安心。

③ ログインが必要なサイトは注意
Cookieを消すとログアウトしがち。ID/パスワードを確認してから。

方法1：履歴とWebサイトデータをまとめて削除（いちばん簡単）

※影響は大きいですが、手順は一番シンプルです。

  1. 設定アプリを開く
  2. 下にスクロールしてSafariを開く
  3. 「履歴とWebサイトデータを消去」をタップ
  4. 確認が出たら「履歴とデータを消去」

これで「キャッシュ」「Cookie」「閲覧履歴」などがまとめて消えることが多いです。
そのぶんログアウトやサイト設定のリセットが起きやすいので注意。

方法2：特定サイトだけ削除（影響を小さく）

「このサイトだけ変」「このサイトだけログインできない」なら、こっちが安全です。

  1. 設定 → Safari
  2. 下の方にある 「詳細」 を開く
  3. 「Webサイトデータ」 を開く
  4. 一覧から該当サイトを探す（上に検索が出る場合もあります）
  5. 右上の「編集」 → 消したいサイトの削除
    または一番下の「すべてのWebサイトデータを削除」（影響大）
注意 “サイトだけ削除”でもログアウトは起きる

WebサイトデータにはCookieも含まれるため、特定サイトだけでもログアウトすることがあります。
ただし、他サイトへの影響は最小で済みます。

キャッシュ/Cookieを消すと何が起きる？（大事）

キャッシュ

画像やページデータの一時保存。壊れると表示崩れや真っ白の原因に。

Cookie

ログイン状態やサイト設定の保存。消すと再ログインが必要になることが多い。

結論

基本は安全。困るのはログインし直しくらいのことが多いです。

「履歴とWebサイトデータを消去」が押せない/出ない

  • スクリーンタイム（制限）で履歴削除が制限されている
  • 閲覧履歴が無い（直前に消した／ほぼ見ていない）
  • 学校/管理端末で設定変更ができない
対処 スクリーンタイムを確認
  1. 設定 → スクリーンタイム
  2. コンテンツとプライバシーの制限
  3. 制限がONなら、Safari関連の制限項目を確認

※管理端末は保護者/管理者側でしか変えられない場合があります。

まだ直らない時にやること

  1. 回線を変える（Wi‑Fi ↔ 4G/5G）
  2. プライベートブラウズで開く（Cookie影響を切り分け）
  3. JavaScriptがOFFになっていないか確認（設定 → Safari）
  4. 広告ブロックを一時OFF（表示崩れ原因になることあり）
  5. 再起動 / OS更新

「特定サイトだけ」なら、まずは方法2（そのサイトだけ削除）が一番おすすめです。
